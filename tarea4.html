Comandos Git y GitHub

Comando y posibles parámetros Breve descripción Ejemplo
configuración
git config −−global user.name “[nombre-apellido]” Establecer un nombre que sea identificable para cuando se revise el historial de versiones git config −−global user.name “Juan Pérez”
git config −−global user.email “[email válido]” Establecer una dirección de correo electrónico que se asociará a cada commit del historial git config −−global user.email “juan.perez@example.com”
git config −−global -e Fichero con la información de configuración de Git git config −−global -e
Inicialización
git init Inicializar un directorio existente como repositorio Git git init
git clone [url] Recuperar un repositorio completo desde una ubicación alojada a través de una URL git clone https://github.com/usuario/repositorio.git
Escenario
git status Mostrar los archivos modificados en el directorio de trabajo, preparados para su próximo commit git status
git add [fichero] Añadir un archivo para el próximo commit (escenario) git add ejemplo.txt
git add -u Actualiza el índice sólo donde ya tiene una entrada que coincide con . git add -u .
git reset [fichero] Desenvolver un archivo conservando los cambios en el directorio de trabajo git reset ejemplo.txt
git reset −−soft HEAD Desenvolver el último commit conservando los cambios en el directorio local git reset −−soft HEAD
git diff Diferencia de lo que se cambia pero no se pone en escena git diff
git diff −−staged Diferencia de lo que se ha puesto en escena pero aún no se ha hecho commit git diff −−staged
git commit -m “[mensaje descriptivo]” Hacer un nuevo commit con su respecta descripción git commit -m "Añadido el archivo ejemplo.txt"
Ramas
git branch Listar las ramas y aparecerá un * al lado de la rama actualmente activa git branch
git branch [nombre de la rama] Crear una nueva rama en el commit actual git branch nueva_rama
git checkout -b [nombre de la rama] Creación y desplazamiento a la rama indicada git checkout -b nueva_rama
git branch -d [nombre de la rama] Elimina la rama indicada git branch -d nueva_rama
git checkout [nombre de la rama] Cambiar a otra rama y comprobarla en el directorio de trabajo git checkout otra_rama
git checkout −− . Retomar el estado del último commit git checkout −− .
git merge [rama] Fusionar el historial de la rama especificada en la actual git merge otra_rama"
git log Mostrar todos los commits en el historial de la rama actual git log"
git log Mostrar el historial de commits de la rama actualmente activa git log
git log ramaB...ramaA Mostrar los commits de la ramaA que no están en la rama B git log ramaB...ramaA
git log −−follow [file] Mostrar los commits que cambiaron el archivo, incluso a través de los renombramientos git log −−follow ejemplo.txt
git diff ramaB...ramaA Mostrar la diferencia de lo que hay en la rama A que no está en la rama B git diff ramaB...ramaA
git show [SHA] Mostrar cualquier objeto en Git en formato legible para el ser humano git show abc123
Seguimiento de los cambios
git rm [fichero] Eliminar el archivo del proyecto y preparar la eliminación para el commit git rm ejemplo.txt
git mv [ruta existente] [nueva ruta] Renombrar un archivo o directorio o cambiar la ruta de un archivo existente y realizar el movimiento git mv ruta_existente/ejemplo.txt nueva_ruta/ejemplo.txt
git log −−stat -M Mostrar todos los registros de commit con indicación de las rutas que se han movido git log −−stat -M
Ignorando los patrones
logs/
*.notes
pattern*/		 Guarda un archivo con los patrones deseados como .gitignore echo -e "logs/\n*.notes\npattern*/" > .gitignore
git config –global core.excludesfile [fichero] Git ignora ese fichero para todos los repositorios git config –global core.excludesfile .gitignore_global
Compartir y Actualizar
git remote add [alias] [url] Añadir una URL de git como alias git remote add origin https://github.com/usuario/repositorio.git
git fetch [alias] Obtener todas las ramas de ese Git remoto git fetch origin
git merge [alias]/[rama] Fusionar una rama remota en su rama actual para actualizarla git merge origin/rama_remota
git push [alias] [rama] Transmitir los commits de la rama local a la rama del repositorio remoto git push origin rama_local"
git pull Obtener y fusionar los commits de la remota a la rama local git pull"
Tags
git tag Visualizar todos los tags git tag
git tag -a [ID del tag] -m "[mensaje descriptivo]" Crear un tag para el último commit realizado con un mensaje enlazado a este tag git tag -a v1.0 -m "Primera versión estable"
git tag -a [ID del tag] [hash del commit] -m "[mensaje descriptivo]" Crear un tag para un commit identificado con su hash git tag -a v0.9 9fceb02 -m "Versión de prueba"
git tag -d [ID del tag] Eliminar el tag específicado git tag -d v0.9"
git push −−tags Enviar todos los tags al repositorio remoto git push −−tags"
Reescribir la Historia
git rebase [rama] Aplicar cualquier commit de la rama actual por delante de la especificada git rebase master"
git reset −−hard [commit] Despejar el escenario y reescribir el árbol de trabajo a partir del commit específicado git reset −−hard 9fceb02"
Commits Temporales
git stash Almacenamiento de los cambios realizados en la "pila de almacenamiento", para centrarse en otro campo y volver más adelante al punto anterior git stash
git stash list Listar el orden de pila de los cambios en los archivos almacenados git stash list
git stash pop Toma un cambio almacenado, lo elimina de la "pila de almacenamiento" y lo aplica al árbol de trabajo actual git stash pop
git stash drop Descartar los cambios de la parte superior de la "pila de almacenamiento" git stash drop
git stash clear Borrar todas las entradas del stash list git stash clear
git stash apply Restaurar el último registro del stash git stash apply
git stash apply [ID del stash] Restaurar el stash identificado con su ID (por ejemplo, ID: stash@{0}) git stash apply stash@{0}
git stash save −−keep-index Guardar todos los archivos menos los archivos del stage git stash save −−keep-index
git stash save −−include-untracked Incluir todos los archivos, incluso a los que git no da seguimiento git stash save −−include-untracked
git stash list −−stat Aparece más información para cada uno de los stash de los registros git stash list −−stat
git stash branch [nombre de la rama] [ID-Stash] Creación de una rama a partir de un stash identificado con su ID git stash branch nueva_rama stash@{0}
git stash show Información detallada del stash git stash show
